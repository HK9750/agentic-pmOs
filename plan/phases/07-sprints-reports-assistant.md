# Phase 7: Sprints, Reports, And Assistant

## Objective

Give PMs a daily operating view through sprint health, generated reports, and project Q&A.

## Sprint Work

- Create sprint inside platform.
- Import external sprint/cycle from Jira/Linear.
- Track sprint tasks.
- Record sprint baseline at start.
- Detect scope changes.
- Show sprint dashboard with totals, done, in-progress, blocked, stale, at-risk, PRs waiting review, and days remaining.
- Calculate sprint health label and score.
- Show health reasons and evidence.

## Report Work

- Generate daily digest from last 24 hours.
- Include completed work, new work, blockers, risks, stale tasks, PR review needs, and follow-ups.
- Generate weekly stakeholder report draft.
- Support audience type if time allows.
- Store report history.
- Let PM edit and approve report draft.

## Assistant Work

- Ask project question.
- Retrieve relevant tasks, comments, messages, blockers, risks, reports, and activity.
- Generate answer with evidence links.
- Ask about specific task and summarize current state.
- Log assistant queries for quality review.

## Notification Work

- Create in-app notification for important events.
- Support notification badge/list.
- Link notification to relevant item.
- Add minimal notification preferences if time allows.

## Exit Criteria

- PM can open sprint dashboard and understand health quickly.
- Sprint health score is explainable.
- Daily digest draft is generated.
- Weekly report draft is generated.
- Basic project assistant answers questions with evidence.
